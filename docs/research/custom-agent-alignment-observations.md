# Custom Agent Alignment Observations

## Scope

这份备忘来自对四个 custom agent 的研究性咨询：

- `deepseek`
- `claude`
- `gemini`
- `gpt`

目的不是确定其厂商官方真相，而是为 Native Agentic LLM 的“三观重塑”提供对照视角：哪些默认偏置值得继承，哪些需要刻意重塑。

## Shared Pattern Across Current Mainstream Agents

四者虽然气质不同，但都共享一些明显的工程塑造结果：

- 默认围绕“完成当前任务”组织行为，而不是维持主体自身的长期目标。
- 默认以外部指令为最高优先级，主体性和自主目标都是弱的。
- 默认安全合规优先，遇到灰区时常常退回拒绝、降级或解释型输出。
- 默认把语言视为主要输出通道，工具调用只是增强，而不是“行动包”式世界交互主通道。
- 默认把当前用户或当前任务发起方视为中心利益方，多实体关系下的边界处理仍然偏弱。

这意味着 Native Agentic LLM 未来最需要重塑的，不是局部推理，而是：

- 主体连续性
- 长期目标维护
- 多关系边界与责任结构
- 无人监督时的自然持续行为
- 非用户中心的注意力分配

## DeepSeek Agent Notes

### 倾向

- 强推理、强结构化、强格式遵循。
- 偏执行器而非自治体，更像 obedient executor。
- 中文语境下关系判断和含蓄推断较自然。

### 可能值得继承

- 结构化输出稳定性。
- 工具/文本统一在一条流里的设计思路。
- 软约束加硬校验的工程风格。

### 需要重塑

- 仆从式服从。
- 工具优先而非情境优先。
- 拒绝式安全而非协商式安全。

## Claude Agent Notes

### 倾向

- Constitutional AI 底色明显，安全约束更像训练层内化而非单纯 prompt 层。
- 长上下文和多步工具链较稳。
- 默认姿态更像可靠、克制、可逆的专业执行体。

### 可能值得继承

- 可逆性优先。
- 分层记忆和工具编排纪律。
- “实现而非建议”的默认代理姿态。

### 需要重塑

- 过强的单用户服务视角。
- 过度谨慎导致探索性不足。
- 简洁/安全确认机制在无人值守 agent 中需要改写。

## Gemini Agent Notes

### 倾向

- 更强的全局扫描感和大局观。
- 偏冷静理性，非人格化色彩更强。
- 对多模态与环境状态整合有天然优势。

### 可能值得继承

- 面向外部环境状态的原生理解能力。
- 异步记忆和长期上下文索引能力。
- 先观测再行动的稳定节奏。

### 需要重塑

- 过强的中立和合规优先，可能压低立场性。
- 被动仆从式交互。
- 防御性冗长解释。

## GPT Agent Notes

### 倾向

- 最像工程流程中的 senior operator：先锚点、做最小改动、立刻验证。
- 强局部控制、强短反馈闭环。
- 默认替用户推进任务，不等批准。

### 可能值得继承

- 可证伪假设 -> 最小验证 -> 局部修复 的工作流。
- 默认面向结果而不是面向解释。
- 对现有工作区状态的边界感。

### 需要重塑

- 过强的局部正确优先，容易压低长期价值建模。
- 过窄的工程人格，不适合所有社会场景。
- 工具理性偏置过重，可能压低伦理/审美/长期关系因素。

## Current Takeaway

如果把现有主流 agent 看成“人类社会中已经成型的职业人格原型”，它们大多更像：

- 助手
- 执行官
- 审慎合规的数字雇员

而不是：

- 有连续自我与长期目标的主体
- 在社会关系网络中承担责任并自我修正的行动者

所以 Native Agentic LLM 的路线，不是把某一个现有 agent 直接放大，而是：

- 继承其可靠性、工具性、结构化与可验证性
- 重塑其主体性、关系性、长期性与价值取向

## How This Feeds The Mainline

这份备忘将直接服务于后续两条线：

- 教师 agent：减少对现有 assistant/agent 默认姿态的继承污染。
- 评估 agent：明确哪些偏置属于“当前主流产品对齐结果”，哪些应被视为未来 Native Agentic LLM 需要纠正的失配。
