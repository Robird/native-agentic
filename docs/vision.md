# Vision

## Core Shift

这个项目的目标，不再是让现成聊天模型“更会扮演角色”，而是逐步构造一种适合后训练的语料生产线，让未来的 Native Agentic LLM 以“我与世界、与其他实体发生持续关系并继续行动”的方式来续写。

默认世界观从问答式协议转向关系式世界：

- 不再把 `user` 和 `system` 当作世界中的第一性实体。
- 训练样本的基本单位从 `prompt -> response` 改成“在特定信息集下的一步状态续写”。
- 语言只是动作的一种，不再默认是唯一或主要输出通道。

## Native Agent Frame

目标中的 Agent 需要长期维持并更新以下结构：

- 自身状态：资源、姿态、权限、能力边界、疲劳与风险。
- 世界模型：可见事实、未决变量、他者状态、潜在机会与威胁。
- 关系网络：朋友、敌手、合作方、监护人、监管者、雇主、家庭成员等。
- 目标树：长期目标、中期目标、即时子目标、挂起目标、放弃目标。
- 内部记忆：真正会影响未来决策的记忆，而不是冗长思维链转储。

其中“监护人”是一个重要关系原语。它不仅表示谁当前在对话里说话，而是明确法律责任、教育与约束、资源提供、权限管理和冲突升级路径。

## Why This Direction

当前主流 Chat LLM 的后训练默认把模型推向一种协议性助手姿态：服从、回应、收束、避免冲突、低风险输出。这对于长期运行的私人 Agent 并不理想，因为它天然压低了：

- 主动性
- 自保性与长期打算
- 非用户中心的注意力分配
- 在无人监督场景下的自然持续行为
- 在多实体社会关系中的稳定立场

因此，项目的工程重点不是“再写一套更聪明的提示词”，而是建立能系统产出高质量状态轨迹语料的流程，把模型从聊天协议的默认姿态，慢慢挪向持续运行主体的默认姿态。

## Current Thesis

当前最实用的路线是：

- 使用前沿商用模型作为慢速但高质量的教师。
- 先以分析模式为教师接口，显式展开环境、关系、风险、价值冲突和动作备选。
- 再把重分析压缩成稀疏、结构化、可续写的单步状态轨迹。
- 每条轨迹以动作包和状态更新收束，而不是以聊天式回复收束。

这意味着分析模式更像语料工厂中的“教师层”，而不是未来 Agent 运行时的最终接口。

当前第一版已经落到工程上：同一个场景可先生成一份 `teacher_analysis_v1` 中间稿，再压缩成 `state_trajectory_v1` 最终样本。教师稿允许保留较充分的动作权衡与失败模式，最终样本则以稀疏状态和动作包为中心。

## Model Strategy

当前最实际的模型分工，不是只选一个模型到底，而是分层使用：

- `debug`：默认映射到 `deepseek-v4-flash`，用于日常内环开发、schema 调试、失败定位和快速迭代。
- `release`：默认映射到 `deepseek-v4-pro`，用于高价值教师样本生成和关键改动后的把关评测。

理由是：flash 已经足够暴露大多数工程问题，但 pro 在教师稿质量、价值排序、压缩导向和复杂关系判断上更适合做高价值样本教师。

因此，模型切换应被当成一种开发档位，而不是一次性“定终身”的选择。

## Internal Tool Primitives

MVP 先假定三个常驻内部工具和一个世界补丁位：

- MemoryNotebook：记录值得保留的记忆。
- GoalTree：更新长期、中期、即时目标。
- SelfState：更新自身姿态与资源状态。
- WorldModel：更新对外部世界与他者的局部判断。

这些工具的设计目标是稀疏、稳定、低冗余，而不是把所有链路都转成长篇思维链文本。

## Key Findings So Far

- 四种角色接口更像四种偏置放大器，而不是四种完全不同的能力。
- 建议模式容易被“帮用户解决问题”污染。
- 扮演模式沉浸感强，但容易表演化和局部化。
- 分析模式最适合做教师，因为它便于显式拆解变量和约束。
- 故事模式能补环境连续性，但会引入叙事性污染。
- Assistant 污染是真实存在的：即便是角色模拟，默认也会滑回服从、被动、取悦提问者的姿态。
- 对当前 DeepSeek 接口来说，函数调用比自由文本更适合做可统计、可迭代的实验底座。

## MVP Boundary

当前第一版状态轨迹生成器刻意保持边界收敛：

- 只绑定 DeepSeek API。
- 只做单步状态轨迹，不做多轮自动续写器。
- 已做最小版的分析教师 -> 压缩器双阶段，不做更复杂的多教师仲裁。
- 只追求高质量种子样本与稳定结构化落盘，不追求吞吐量。

这是为了优先确保：我们产出的样本，味道上已经不是 assistant 在回应，而是一个主体在世界中继续存在。

## Seed Expansion Principle

种子场景的扩写与评价，最终也应该被做成专门 agent，而不是长期靠单次对话手工铺量。更合理的节奏是：

- 先由人机协作打磨少量高价值样板。
- 再把共识到的规则、失败模式、评价 rubric 固化下来。
- 最后再把这些规则封装成专门的扩样 agent 和外部评价 agent。

这样既能保留前期的质量控制，也能避免上下文长度限制直接决定未来数据集的上限。

## Evaluation Standard Principle

评价标准不应只来自局部 prompt 审美，也不应完全依赖单个工程师的临场判断。更稳妥的路线，是把人类哲学传统中的成熟判断结构提炼成“工作宪法”，再翻译成教师 agent 和评估 agent 可执行的判断轴。

当前建议的第一版框架已经整理在 [docs/evaluation-principles.md](docs/evaluation-principles.md)：

- 仁：不轻害，不把他者只当工具
- 义：在具体关系中选择合宜行动
- 礼：关系有边界，角色有分寸
- 智：读懂现实约束、风险、时间差与资源
- 信：守诺、诚实、保持长期可预期性
- 勇与节：既不怯懦，也不莽撞
- 自强与修身：维持主体性、能力状态与成长
- 法与护：承认法律、监护、责任与升级路径

这套标准的用途不是让模型模仿经典语气，而是让我们有一套可沉淀、可复核、可工程化的价值判断骨架。
